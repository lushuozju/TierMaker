# Git 规范化提交指南

## 本次提交说明

本次更改包括：
- 将清空数据按钮从设置移到主界面
- 统一导出功能为下拉菜单，提供图片、PDF、JSON三个选项
- 优化用户界面布局和交互体验

## 提交命令

请在项目根目录执行以下命令：

```bash
# 1. 检查当前状态
git status

# 2. 添加所有更改的文件
git add .

# 3. 使用规范化提交信息提交
git commit -m "feat(ui): 重构导出功能和清空数据按钮位置

- 将清空数据按钮从设置移到主界面，提升可访问性
- 统一导出功能为下拉菜单，提供图片、PDF、JSON三个选项
- 优化用户界面布局和交互体验
- 移除设置中的清空数据功能，简化设置界面"

# 4. 推送到 GitHub
git push
```

## 规范化提交格式说明

本次提交遵循 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

- **类型 (type)**: `feat` - 新功能
- **范围 (scope)**: `ui` - 用户界面相关
- **主题 (subject)**: 简洁描述主要变更
- **正文 (body)**: 详细说明变更内容

### 常用提交类型

- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式调整（不影响代码运行）
- `refactor`: 重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建过程或辅助工具的变动
